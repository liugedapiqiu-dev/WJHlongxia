## 📐 设计原则（来自 automation-workflows 技能）

1. **先优化流程，再自动化** - 表格设计前先理清业务流程
2. **从简单开始** - 核心字段优先，扩展字段后续添加
3. **必须设置错误处理** - 自动化失败要有通知
4. **记录文档** - 每个表格都要有使用说明

## 📊 表 1：采购管理表

### 业务场景
健豪需要管理 17 家供应商的采购订单，当前痛点：
- 口头沟通多，缺乏书面记录
- 交期跟踪困难
- 价格波动没有历史记录

### 字段设计

| 字段名 | 字段类型 | 必填 | 说明 | 示例 |
|--------|----------|------|------|------|
| 采购单号 | 自动编号 | ✅ | PO-YYYYMMDD-001 | PO-20260304-001 |
| 供应商 | 单向关联 | ✅ | 关联供应商管理表 | 深圳 XX 包装厂 |
| 产品名称 | 单行文本 | ✅ | | 高尔夫毛巾外箱 |
| SKU | 单行文本 | ✅ | | GT-BOX-001 |
| 采购数量 | 数字 | ✅ | | 500 |
| 单价 | 数字（货币）| ✅ | 含税价 | ¥3.50 |
| 总金额 | 公式 | 🔒 | 数量 × 单价 | ¥1,750 |
| 下单日期 | 日期 | ✅ | | 2026-03-04 |
| 要求交期 | 日期 | ✅ | | 2026-03-15 |
| 实际交期 | 日期 | | 到货后填写 | |
| 交期状态 | 单选 | 🔒 | 公式计算 | 🟢 正常/🟡 延期/🔴 严重延期 |
| 付款条件 | 单选 | ✅ | 月结 30 天/货到付款/预付 30% | 月结 30 天 |
| 付款状态 | 单选 | | 未付款/部分付款/已付款 | 未付款 |
| 质检状态 | 单选 | | 待质检/合格/不合格 | 待质检 |
| 入库状态 | 单选 | | 未入库/部分入库/已入库 | 未入库 |
| 备注 | 多行文本 | | | 加急，用于蜘蛛侠书包项目 |

### 自动化规则

| 触发器 | 条件 | 执行动作 | 错误处理 |
|--------|------|----------|----------|
| 创建新采购单 | - | 发送飞书消息给健豪："新采购单 PO-xxx 已创建" | 失败则记录到日志表 |
| 当前日期 > 要求交期 | 实际交期为空 | 1. 交期状态变为"🟡 延期"<br>2. 发送提醒给健豪 | - |
| 当前日期 > 要求交期 +7 天 | 实际交期为空 | 1. 交期状态变为"🔴 严重延期"<br>2. 发送加急消息 | - |
| 质检状态变为"不合格" | - | 1. 发送消息给健豪<br>2. 创建任务："处理不合格品" | - |

### 视图设计
- **表格视图**：默认视图，所有字段
- **看板视图**：按"交期状态"分组（正常/延期/严重延期）
- **日历视图**：显示"要求交期"，方便查看每周到货情况
- **仪表盘**：
  - 本月采购总额
  - 延期订单数量
  - 供应商交期准确率排名

## 📊 表 2：库存管理表

### 业务场景
- 高尔夫毛巾、游戏垫、台钳等多产品线
- FBA 仓 + 国内仓 + 在途库存
- 需要避免断货和积压

### 字段设计

| 字段名 | 字段类型 | 必填 | 说明 | 公式/示例 |
|--------|----------|------|------|-----------|
| SKU | 单行文本 | ✅ | 唯一标识 | GT-TOWEL-001 |
| 产品名称 | 单行文本 | ✅ | | 高尔夫毛巾 |
| 产品类目 | 单选 | ✅ | 高尔夫/玩偶/书包/水疗 | 高尔夫 |
| 仓库 | 单选 | ✅ | FBA 美仓/FBA 欧仓/深圳仓/在途 | FBA 美仓 |
| 当前库存 | 数字 | ✅ | 实时库存 | 250 |
| 在途库存 | 数字 | | 已发货未入库 | 500 |
| 可售库存 | 公式 | 🔒 | 当前 + 在途 - 预留 | 750 |
| 预留数量 | 数字 | | 已下单未发货 | 0 |
| 日均销量 | 数字 | | 过去 30 天平均 | 15 |
| 可售天数 | 公式 | 🔒 | 可售库存 ÷ 日均销量 | 50 天 |
| 安全库存 | 公式 | 🔒 | 日均销量 × 安全天数 (15) | 225 |
| 补货点 | 公式 | 🔒 | 安全库存 + 在途 | 725 |
| 建议补货量 | 公式 | 🔒 | (采购周期 + 入库天数) × 日均销量 - 在途 - 当前 | 见下方 |
| 库龄 | 日期 | | 首次入库日期 | 2026-01-15 |
| 滞销标记 | 复选框 | 🔒 | 可售天数 > 90 自动勾选 | ☐ |
| 状态 | 单选 | 🔒 | 正常/建议补货/紧急补货/滞销 | 正常 |

**建议补货量公式详解**：
```
IF(可售天数 < 安全天数，
  (采购周期 30 天 + 入库天数 20 天) × 日均销量 - 在途库存 - 当前库存，
  0
)
```

### 自动化规则

| 触发器 | 条件 | 执行动作 | 错误处理 |
|--------|------|----------|----------|
| 可售天数 < 安全天数 (15) | - | 1. 状态变为"建议补货"<br>2. 发送飞书消息 | - |
| 可售天数 < 安全天数 × 50% | - | 1. 状态变为"紧急补货"<br>2. 发送加急消息<br>3. 创建采购申请 | 失败则记录日志 |
| 可售天数 > 90 | - | 1. 滞销标记勾选<br>2. 状态变为"滞销"<br>3. 发送清理建议 | - |
| 库存 < 0 | - | 1. 发送超卖警报<br>2. 创建紧急任务 | - |

### 视图设计
- **表格视图**：默认视图
- **看板视图**：按"状态"分组（正常/建议补货/紧急补货/滞销）
- **仪表盘**：
  - 总库存价值（各 SKU 成本 × 数量）
  - 需要补货的 SKU 数量
  - 滞销 SKU 列表
  - 库存周转天数分布

## 📊 表 3：供应商管理表 (SRM)

### 业务场景
17 家供应商需要分级管理，优胜劣汰。

### 字段设计

| 字段名 | 字段类型 | 必填 | 说明 | 示例 |
|--------|----------|------|------|------|
| 供应商 ID | 自动编号 | ✅ | SUP-001 | SUP-001 |
| 供应商名称 | 单行文本 | ✅ | | 深圳 XX 包装厂 |
| 供应商类型 | 单选 | ✅ | 工厂/贸易商/服务商 | 工厂 |
| 主营类目 | 多选 | ✅ | 包装/ textiles/电子/塑胶 | 包装，印刷 |
| 联系人 | 单行文本 | ✅ | | 张经理 |
| 联系电话 | 电话 | ✅ | | 138xxxxxxx |
| 合作状态 | 单选 | ✅ | 合作中/暂停/已淘汰 | 合作中 |
| 合作开始日期 | 日期 | | | 2024-06-01 |
| 合同到期日 | 日期 | | | 2026-12-31 |
| 综合评分 | 数字 (1-5) | 🔒 | 加权平均 | 4.2 |
| 交期准确率 | 百分比 | 🔒 | 准时交货单数 ÷ 总单数 | 85% |
| 质量合格率 | 百分比 | 🔒 | 合格批次 ÷ 总批次 | 98% |
| 价格竞争力 | 单选 | | 高/中/低 | 中 |
| 配合度评分 | 数字 (1-5) | | 主观评价 | 4 |
| 月采购额 | 数字 (货币) | | 过去 3 个月平均 | ¥50,000 |
| 付款方式 | 单选 | | 月结/货到付款/预付 | 月结 30 天 |
| 备注 | 多行文本 | | | 配合度高，可加急 |

### 评分计算公式
```
综合评分 = (交期准确率 × 0.4 + 质量合格率 × 0.4 + 配合度评分/5 × 0.2) × 5
```

### 自动化规则

| 触发器 | 条件 | 执行动作 |
|--------|------|----------|
| 合同到期日 < 30 天 | - | 发送提醒："供应商 xxx 合同即将到期" |
| 综合评分 < 3.0 | - | 状态建议改为"暂停"，发送评估提醒 |
| 交期准确率 < 70% | - | 发送警告："供应商 xxx 交期准确率过低" |
| 新增采购单 | 关联此供应商 | 更新"月采购额"滚动平均值 |

### 视图设计
- **看板视图**：按"合作状态"分组
- **甘特图**：合同时间线，显示到期日
- **仪表盘**：
  - 供应商评分分布
  - 交期准确率 Top 5 / Bottom 5
  - 合同到期提醒（30 天内）

## 📊 表 4：FBA 补货计算表

### 业务场景
专门针对亚马逊 FBA 仓库的补货计算，考虑头程时效。

### 字段设计

| 字段名 | 字段类型 | 公式/说明 |
|--------|----------|-----------|
| SKU | 单行文本 | |
| 产品名称 | 单行文本 | |
| 目标仓库 | 单选 | FBA 美国西部/FBA 美国东部/FBA 欧洲 |
| 日均销量 | 数字 | 过去 30 天平均，可手动调整 |
| 销售趋势 | 单选 | 上升/平稳/下降（根据 7 天 vs 30 天） |
| 采购周期 | 数字 | 默认 30 天（下单到生产完成）|
| 头程时效 | 数字 | 海运 35 天/空运 7 天/快递 5 天 |
| 入库天数 | 数字 | FBA 收货 + 上架，默认 5 天 |
| 总补货周期 | 公式 | 采购周期 + 头程时效 + 入库天数 |
| 安全天数 | 数字 | 默认 15 天（缓冲）|
| 安全库存 | 公式 | 日均销量 × 安全天数 |
| FBA 在库 | 数字 | 从亚马逊 API 或手动更新 |
| FBA 在途 | 数字 | 已发货未入库 |
| 预留数量 | 数字 | 顾客已下单未发货 |
| 可售库存 | 公式 | FBA 在库 + FBA 在途 - 预留 |
| 可售天数 | 公式 | 可售库存 ÷ 日均销量 |
| 补货点 | 公式 | 安全库存 + FBA 在途 |
| 建议补货量 | 公式 | 见下方 |
| 补货状态 | 单选 | 无需补货/建议补货/紧急补货 |
| 最后补货日期 | 日期 | |
| 建议发货方式 | 公式 | 根据紧急程度推荐海运/空运/快递 |

**建议补货量公式**：
```
IF(可售天数 < 安全天数，
  总补货周期 × 日均销量 - FBA 在途 - FBA 在库，
  0
)
```

**建议发货方式公式**：
```
IF(可售天数 < 7, "快递 (5 天)",
  IF(可售天数 < 15, "空运 (7 天)",
    "海运 (35 天)"
  )
)
```

### 自动化规则

| 触发器 | 条件 | 执行动作 |
|--------|------|----------|
| 可售天数 < 安全天数 | - | 1. 补货状态 = "建议补货"<br>2. 发送消息 |
| 可售天数 < 7 | - | 1. 补货状态 = "紧急补货"<br>2. 发送加急消息<br>3. 建议发货方式 = 快递 |
| 创建补货建议 | - | 自动在采购管理表创建草稿记录 |

| 发货日期 | 日期 | |
| 预计到达 | 日期 | |
| 实际到达 | 日期 | |
| 物流轨迹 | 多行文本 | 关键节点记录 |
| 当前状态 | 单选 | 已发货/运输中/清关中/派送中/已签收/异常 |

| 运费 | 数字（货币）| |

| 触发器 | 条件 | 执行动作 |

## 📋 实施优先级（按 ROI 排序）

根据 automation-workflows 的 ROI 计算：

| 表格 | 预估时间节省/月 | 实施难度 | 优先级 |
|------|----------------|----------|--------|
| 库存管理表 | 10 小时（避免断货 + 减少积压）| 中 | ⭐⭐⭐⭐⭐ |
| FBA 补货计算表 | 8 小时（自动计算补货量）| 中 | ⭐⭐⭐⭐⭐ |
| 采购管理表 | 6 小时（减少沟通成本）| 低 | ⭐⭐⭐⭐ |
| 供应商管理表 | 4 小时（评估更高效）| 低 | ⭐⭐⭐ |

1. 先做 **库存管理表** + **FBA 补货计算表**（最紧急，避免断货）
